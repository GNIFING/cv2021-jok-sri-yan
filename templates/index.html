About

โปรเจคนี้อยู่ในปัญหาที่เรียกว่า Pose Classification โดยที่เราจะนำท่าทางของผู้เล่น มาจำแนกเป็น Control ท่าต่างๆ ที่ใช้ภายในเกมประเภท Subway Surfer โดยการนำ Pose Estimation Landmarks ของ MediaPipe มาจำแนกเป็นท่าต่างๆ



Poses

1. เลี้ยวซ้าย: ยกมือซ้ายให้เหนือไหล่



2. เลี้ยวขวา: ยกมือขวาให้เหนือไหล่



3. กระโดด: กระโดดโดยไม่ย่อเข่า



4. ย่อ (กลิ้ง): ย่อลำตัวเสมือนกำลังจะนั่งยอง




Mechanism

1. นำรูปที่ได้จากกล้องมาเข้า MediaPipe แบบ PoseEstimation โดยจะได้ Landmark ทั้งหมด 33 จุด ที่มีพิกัด x y z



2. นำพิกัด x y z จากทั้ง 33 Landmarks มาคัดกรองผ่านระบบ Rule-based System เพื่อจำแนกเป็นท่าทางทั้ง 4 ท่า



3. นำท่าทางที่ Classify เสร็จแล้ว มา Control ตัวเกม ผ่าน Python Library ชื่อ Pynput




Evaluation

นำ Rule-based Classifier ที่เขียนขึ้น มาทดสอบด้วยรูปภาพที่ถ่ายกันเองทั้ง 3 ท่า (ไม่รวมท่ากระโดด เพราะท่ากระโดดมีการคิดกลไกแบบเทียบกับ Frame ก่อนหน้า) รวมกว่า 300 รูป ซึ่งได้ Accuracy เป็น 0.9831 โดย Classify ผิดเพียง 6 ท่า (5 ท่าในนี้มีความสมเหตุสมผลที่ Classify ผิด)




Alternative: ML Pose Classifier

มีการใช้ Machine Learning Model มาทดลองทำ Classifier ซึ่งได้ทดลอง 2 Model คือ XGBoost และ KNN แล้วใช้ Dataset ที่นำไป Evaluate ระบบ Rule-based ก่อนหน้านี้ มาแบ่ง 70:30 เพื่อนำมา Train จากนั้นนำไป Evaluate บน Test set 30% ที่แยกไว้ ได้ผลดังรูปทั้งคู่ คือ Accuracy 100%

ทั้งนี้ Model ไม่ได้มีการจำแนกท่าอยู่นิ่ง (Idle) แสดงว่าจะต้องใช้ Probability ในแต่ละท่ามาทำ Threshold ซึ่งจำเป็นต้องไปปรับเพิ่มเติม

นอกจากนี้ เหตุผลที่ยังคงใช้ Rule-based เช่นเดิม เป็นเพราะว่า Rule-based ที่ได้นั้น มี Accuracy ที่ค่อนข้างดีอยู่แล้ว ซึ่งน่าจะเหมาะสำหรับความ Real-time ที่ต้องใช้ในการเล่นเกมมากกว่าการใช้ ML
